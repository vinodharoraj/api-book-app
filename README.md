**Title:** A simplified library management sytem exposing REST APIs to manage books and authors.

**Project Overview**

Library Book & Author API: a RESTful Spring Boot service providing CRUD operations, relationships and search functionality for managing books and authors.
Built with Java 17, Spring Boot, Spring Data JPA and H2 DB, focused on testability, documentation and easy deployment.

**Requirements / Prerequisites**
  Java JDK 17 installed and JAVA_HOME set. 
  
    Verify: _java -version_
  Maven (recommended 3.8+) installed. 
  
    Verify: _mvn -v_
  Git installed and a GitHub account. 
  
    Verify: _git --version_
  Authentication for pushing to GitHub: 
  
    _Personal Access Token (PAT), SSH key, or GitHub CLI (gh auth login)_
  IntelliJ IDEA 2025.3.2 (recommended) — 
  
    _import project as Maven, enable annotation processors, install Lombok plugin_
  Lombok: 
  
    _IDE plugin + annotation processing enabled (used in the project)_
  H2 database available by default (in-memory) — 
  
    _no DB setup needed for default run_
 If using an external DB (Postgres/MySQL):
 
      _Install the DB and JDBC driver_
  Configure connection in 
  
      _src/main/resources/application.properties _

**Useful commands:**

Run locally: 

_mvn spring-boot:run or java -jar target/*.jar_

Run tests: 

_mvn test (JaCoCo report generated by the build)_


**Quickstart**
    Clone, set up remote if needed, build and run commands.
      Example commands:

            git clone https://github.com/vinodharoraj/api-book-app.git
            cd api-book-app
            mvn clean package
            mvn spring-boot:run
            # or
            java -jar target/api-book-application-0.0.1-SNAPSHOT.jar


**API Endpoints**

    Book Controller
            1. GET /api/books/getBook/{id}
            2. GET /api/books/getAllBooks
            3. GET /api/books/filterBooks
            4. POST /api/books/addBook
    Author Controller
            1.  GET /api/books/authors/search
            2.  PUT /api/books/authors/update/{id}

**Additional Info:** 

  ### Postman Collection

- The Postman collection is included in the repository at
  `src/main/resources/Library Application.postman_collection.json`.
- Start the application locally:
  - `mvn spring-boot:run` or `java -jar target/*.jar`
  - Default base URL: `http://localhost:8080`

- Import into Postman:
  1. In Postman click **Import** → **Files** and select `src/main/resources/Library Application.postman_collection.json`, or drag-and-drop the file into Postman.
  2. The collection will appear in your Collections panel.

- Configure environment:
  - Create an environment (e.g., `Local`) with a variable `baseUrl` set to `http://localhost:8080`.

- Run requests:
  - Open the collection and send individual requests or use the Collection Runner to execute a suite of tests.

- Troubleshooting:
  - If requests fail, confirm the app is running and `baseUrl` matches the server port.
  - If the collection file path or name differs, update the README accordingly.
 
**API Specification**

  An OpenAPI 3.0 YAML specification `api-book-application-schema-v1.0.yaml` is included in the repository at `src/main/resources/api-book-application-schema-v1.0.yaml.` 
  Use it for quick understanding and to generate interactive docs (import into `Swagger Editor`, `Swagger UI`, or `Redoc`).
